# Internationalization (i18n) - AR/EN

## 1. Zero Hardcoding
- **Strict Rule:** No hardcoded strings in the UI. 
- **Implementation:** Use `next-intl` (or chosen library) for all text.
- **Keys:** Use descriptive keys (e.g., `t('auth.login.submit_button')`).

## 2. Multi-Language Files
- **Paths:** Always update `messages/en.json` and `messages/ar.json` simultaneously when adding text.
- **Arabic First:** Ensure the UI supports Right-to-Left (RTL) layouts.

## 3. RTL Styling (Tailwind)
- **Logical Properties:** Use Tailwind logical properties for spacing/positioning.
  - Use `ps-*` (padding-start) instead of `pl-*`.
  - Use `me-*` (margin-end) instead of `mr-*`.
  - Use `start-0` instead of `left-0`.


## 4. Localization Logic
- **Dates:** Use `date-fns` with the appropriate locale (`ar-SA` or `en-US`).
- **Currency:** Use a helper function `formatCurrency(amount, currency)` that uses `Intl.NumberFormat`. Never manually type "SAR".
- **Numbers:** In AR mode, ensure the UI uses Eastern Arabic numerals (١٢٣).
